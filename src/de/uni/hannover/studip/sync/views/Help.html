Beschreibung

Die Stud.IP-FileSync Anwendung lädt alle Dokumente Ihrer Stud.IP Veranstaltungen in einen Ordner auf Ihrem Computer herunter.
Dabei bleibt die Ordnerstruktur der Veranstaltungen erhalten.

Erste Schritte

  1. Starten Sie die Stud.IP-FileSync Anwendung und klicken Sie auf "Sync". Wenn Sie sich noch nicht eingeloggt haben, werden Sie zum Login weitergeleitet.
  2. Es erscheint eine kurze Beschreibung des Loginvorgangs, mit einem Klick auf "Weiter" wird die Stud.IP Login-Seite geladen.
  3. Melden Sie sich nun mit Ihren Accountdaten an und bestätigen Sie den Zugriff auf Ihre Veranstaltungen, danach werden Sie automatisch zum Setup weitergeleitet.
  4. Für eine ausführliche Beschreibung lesen Sie den Abschnitt Einstellungen.
  5. Nach der Einrichtung klicken Sie erneut auf "Sync", um das Herunterladen zu starten.

Sync

  • Die Synchronisation läuft in zwei Schritten ab.
    Zuerst wird nach neuen und geänderten Dokumenten gesucht, die empfangenen Informationen werden in einer lokalen Datei gespeichert. Danach beginnt das Herunterladen der Veranstaltungsdokumente.
  • Möchten Sie die Aktualisierung bzw. das Herunterladen abbrechen, können Sie die Anwendung einfach beenden. Beim Herunterladen werden aktive Downloads zuerst abgeschlossen.
  • Nachdem Sie sich am Anfang eines Semesters in neue Stud.IP Veranstaltungen eingetragen haben, muss mit der Anwendung einmalig nach neuen Veranstaltungen gesucht werden. Klicken Sie hierfür auf "Hilfe" und dann "Neue Veranstaltungen suchen".

Neue Dokumente

Im Menü "Datei" unter "Neue Dokumente" finden Sie eine Übersicht aller Dokumente. Mit einem Doppelklick auf einen Eintrag wird das Dokument geöffnet.

Export Materialsammlung

Beim Exportieren werden alle Veranstaltungsordner in einen Exportordner mit Materialsammlungsstruktur kopiert. Sobald das Kopieren fertig ist, wird der Exportordner geöffnet.

Einstellungen

Die Einstellungen können jederzeit im Menü "Datei" unter "Einstellungen" geändert werden.

  1. Ziel Ordner: Alle Dokumente werden in diesem Ordner gespeichert. Am besten erstellen Sie einen neuen Ordner z.B. "FileSync".
  2. Ordnerstruktur: Diese Einstellung gibt an welche Struktur beim Erstellen der Veranstaltungsordner benutzt werden soll. Sie können eine vordefinierte Struktur wählen oder Ihre eigene eintragen. Wird die Ordnerstruktur geändert, müssen alle Dokumente neu heruntergeladen werden.
    • Vorlesung und Übung zusammenführen:
      Es wird versucht Vorlesungen und Übungen anhand ihres Namens in einem gemeinsamen Veranstaltungsordner mit dem Namen der Vorlesung zu speichern.
      In diesem Ordner wird für jeden Veranstaltungstyp ein Unterordner erstellt ("vorlesung", "uebung", usw.). Sollte dies nicht möglich sein, werden die Veranstaltungen einzeln gespeichert.

      Beispiel:
      SoSe 2015/Analysis A/vorlesung und SoSe 2015/Analysis A/uebung
    • Alle Veranstaltungen einzeln:
      Jede Veranstaltung (Vorlesung, Übung usw.) wird in einem eigenem Veranstaltungsordner gespeichert.

      Beispiel:
      SoSe 2015/Analysis A und SoSe 2015/Übung: Analysis A
    • Materialsammlung:
      Veranstaltungen werden nach ihrem Namen, dem jeweiligen Semester und dem Typ sortiert.

      Beispiel:
      Analysis A/15ss/vorlesung und Analysis A/15ss/uebung
  3. Semester: Sie können auswählen, ob nur die Dokumente des aktuellen Semesters oder aller über Stud.IP sichtbaren Semester heruntergeladen werden sollen.
  4. Geänderte Dateien: Werden Dateien in Stud.IP geändert / aktualisiert, kann die alte Version durch die neue ersetzt oder umbenannt werden. Die aktuellste Version trägt immer den ursprünglichen Namen. Alte Versionen werden durch eine fortlaufende Nummerierung gekennzeichnet.
  5. Leerzeichen ersetzen: Es können automatisch in Datei- und Ordnernamen auftretende Leerzeichen durch einen Binde- oder Unterstrich ersetzt werden. Diese Option soll den Zugriff über die Konsole vereinfachen.

Deinstallieren / Einstellungen löschen

Die Stud.IP-FileSync Anwendung legt einen (versteckten) Ordner ".studip-sync" in Ihrem Benutzerverzeichnis an. Dieser Ordner enthält alle Einstellungen und Logininformationen (OAuth-Token). Ihr Passwort wird nicht gespeichert.
